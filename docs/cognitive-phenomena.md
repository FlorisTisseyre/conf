# cognitive-phenomena.md

# IA = Ingénieur Augmenté / Ingénieur Aliéné

## Phénomènes cognitifs que l'expérience cherche à rendre visibles

---

# Objectif du document

Ce document recense les phénomènes cognitifs que l'atelier cherche à mettre en évidence.

L'objectif n'est pas de prouver que l'IA est bénéfique ou néfaste.

L'objectif est d'observer comment différents modes d'interaction avec l'IA modifient :

- la compréhension ;
- l'apprentissage ;
- la mémorisation ;
- la capacité de raisonnement ;
- l'autonomie ;
- la vitesse d'exécution.

Ces phénomènes constituent le cœur intellectuel de la conférence.

---

# Production vs Compréhension

## Description

Un individu peut produire un résultat sans nécessairement comprendre le système qui a permis de le produire.

L'IA rend ce phénomène beaucoup plus fréquent qu'auparavant.

Un développeur peut :

- générer du code ;
- corriger un bug ;
- implémenter une fonctionnalité ;

sans construire le modèle mental correspondant.

---

## Manifestation observable

Le participant :

- livre rapidement ;
- semble efficace ;
- mais peine à expliquer son implémentation ;
- peine à l'adapter lorsque les règles changent.

---

## Question clé

Le participant possède-t-il :

- une solution ?

ou

- une compréhension ?

---

# Illusion de compréhension

## Description

L'humain confond parfois :

- reconnaissance ;
- compréhension.

Lire une explication produite par une IA donne souvent une sensation de compréhension immédiate.

Cette sensation peut être trompeuse.

---

## Manifestation observable

Le participant affirme :

"Oui, je comprends."

Puis échoue à :

- expliquer ;
- enseigner ;
- modifier ;
- reconstruire.

---

## Question clé

Comprend-il réellement ?

Ou reconnaît-il simplement un schéma familier ?

---

# Cognitive Offloading

## Description

L'être humain délègue une partie de sa charge cognitive à un outil.

Ce phénomène existe déjà avec :

- l'écriture ;
- les calculatrices ;
- les GPS ;
- les moteurs de recherche.

L'IA étend massivement ce phénomène aux tâches intellectuelles complexes.

---

## Effets positifs

- vitesse ;
- réduction de la charge mentale ;
- accès à davantage d'informations ;
- augmentation du débit de production.

---

## Effets négatifs potentiels

- moindre rétention ;
- moindre apprentissage ;
- dépendance accrue à l'outil.

---

## Question clé

Quelles tâches doivent être déléguées ?

Quelles tâches doivent rester cognitives ?

---

# Dette cognitive

## Description

Équivalent cognitif de la dette technique.

L'IA permet parfois de gagner du temps immédiatement.

Mais ce gain peut être payé plus tard :

- en maintenance ;
- en compréhension ;
- en transfert de connaissances ;
- en onboarding.

---

## Manifestation observable

Tout fonctionne aujourd'hui.

Personne ne comprend pourquoi dans trois semaines.

---

## Question clé

La vitesse obtenue aujourd'hui crée-t-elle un coût futur ?

---

# Cartographie mentale du système

## Description

Un ingénieur construit progressivement une représentation interne du système.

Cette cartographie permet :

- l'anticipation ;
- le diagnostic ;
- la maintenance ;
- l'évolution.

L'IA peut soit accélérer cette construction, soit la court-circuiter.

---

## Manifestation observable

Lorsque la règle métier change :

certains participants savent immédiatement où intervenir.

D'autres non.

---

## Question clé

Le participant possède-t-il un modèle mental du système ?

---

# Illusion d'expertise

## Description

L'IA permet de produire des résultats qui ressemblent à ceux d'un expert.

Cela peut créer une confusion entre :

- produire un artefact ;
- posséder une expertise.

---

## Manifestation observable

Le participant obtient un résultat crédible.

Mais ne maîtrise pas :

- les hypothèses ;
- les compromis ;
- les limites.

---

## Question clé

Le résultat est-il expert ?

Ou seulement son apparence ?

---

# Dépendance à l'outil

## Description

Lorsqu'une compétence est systématiquement externalisée, une dépendance peut émerger.

La question n'est pas :

"Est-ce mauvais ?"

La question est :

"Que se passe-t-il lorsque l'outil disparaît ?"

---

## Manifestation observable

Le participant est bloqué sans IA.

---

## Question clé

L'outil augmente-t-il l'autonomie ?

Ou remplace-t-il l'autonomie ?

---

# Apprentissage génératif

## Description

L'IA peut également devenir un accélérateur d'apprentissage.

Elle peut :

- expliquer ;
- questionner ;
- proposer des contre-exemples ;
- jouer le rôle de mentor ;
- adapter son niveau pédagogique.

---

## Manifestation observable

Le participant comprend plus vite qu'il n'aurait compris seul.

---

## Question clé

L'IA remplace-t-elle l'effort ?

Ou améliore-t-elle l'effort ?

---

# Exploration cognitive augmentée

## Description

L'IA permet d'explorer davantage d'alternatives qu'auparavant.

Un ingénieur peut comparer :

- plusieurs architectures ;
- plusieurs algorithmes ;
- plusieurs designs.

Cette capacité peut enrichir considérablement la réflexion.

---

## Manifestation observable

Le participant évalue plusieurs solutions avant d'en choisir une.

---

## Question clé

L'IA est-elle utilisée pour confirmer une idée ?

Ou pour explorer plusieurs possibilités ?

---

# Raisonnement externalisé

## Description

Le raisonnement lui-même peut être progressivement délégué.

Le danger n'est pas la génération de code.

Le danger est la délégation du jugement.

---

## Manifestation observable

Le participant applique une recommandation sans être capable de justifier pourquoi.

---

## Question clé

Qui a réellement pris la décision ?

Le participant ou l'IA ?

---

# Compréhension transférable

## Description

Le véritable apprentissage est transférable.

Une connaissance est acquise lorsqu'elle peut être réutilisée dans un contexte nouveau.

---

## Manifestation observable

Après une modification surprise :

certains participants s'adaptent rapidement.

D'autres repartent de zéro.

---

## Question clé

Le participant a-t-il appris une solution ?

Ou un principe ?

---

# Friction productive

## Description

Certaines difficultés sont nécessaires à l'apprentissage.

Supprimer toute friction peut empêcher la construction de compétences durables.

---

## Manifestation observable

Le participant progresse très vite.

Mais retient très peu.

---

## Question clé

Quelles frictions doivent être éliminées ?

Quelles frictions doivent être préservées ?

---

# Le paradoxe central

L'IA peut simultanément :

- augmenter l'apprentissage ;
- diminuer l'apprentissage.

- augmenter l'autonomie ;
- diminuer l'autonomie.

- augmenter la compréhension ;
- diminuer la compréhension.

- augmenter la qualité ;
- diminuer la qualité.

La différence ne provient pas uniquement du modèle utilisé.

Elle provient principalement :

- de l'intention ;
- du mode de collaboration ;
- du niveau d'esprit critique ;
- de la stratégie cognitive adoptée par l'utilisateur.

---

# Hypothèse finale de la conférence

Le futur ne sera probablement pas une opposition entre :

- humains ;
- intelligences artificielles.

Le futur opposera davantage :

- les personnes qui utilisent l'IA pour enrichir leur cognition ;
- les personnes qui utilisent l'IA pour remplacer leur cognition.

L'enjeu principal n'est donc pas l'adoption de l'IA.

L'enjeu principal est la maîtrise consciente de l'investissement cognitif.

# Chapitre 2 — Phénomènes cognitifs collectifs

# Pourquoi un chapitre collectif ?

La plupart des discussions autour de l'IA se concentrent sur l'individu :

- vais-je apprendre ?
- vais-je devenir dépendant ?
- vais-je rester compétent ?

Pourtant, le développement logiciel est une activité profondément collective.

Une équipe n'est pas simplement la somme de plusieurs cerveaux.

L'IA modifie également :

- les interactions ;
- les mécanismes de décision ;
- le partage de connaissances ;
- les dynamiques de pouvoir ;
- la culture technique.

Certaines conséquences apparaissent uniquement à l'échelle de l'équipe.

---

# Externalisation collective de la connaissance

## Description

Traditionnellement, une partie de la connaissance du système est répartie dans l'équipe.

Chaque membre possède :

- une expertise ;
- un contexte ;
- une mémoire partielle.

L'IA introduit un nouveau dépôt de connaissances externe.

Progressivement, les réponses ne sont plus recherchées dans l'équipe mais auprès de l'outil.

---

## Manifestation observable

Moins de questions aux collègues.

Plus de questions aux LLM.

---

## Question clé

L'équipe apprend-elle encore collectivement ?

Ou délègue-t-elle progressivement sa mémoire à un système externe ?

---

# Diffusion de responsabilité

## Description

Lorsqu'une décision provient d'un humain, son auteur est généralement identifiable.

Avec l'IA, l'origine d'une décision devient plus floue.

On observe parfois :

- "Claude l'a proposé."
- "GPT l'a recommandé."
- "L'agent a généré cette implémentation."

---

## Manifestation observable

Les décisions deviennent plus difficiles à attribuer.

---

## Question clé

Qui est responsable d'une décision prise avec l'aide d'une IA ?

---

# Homogénéisation des solutions

## Description

Les modèles sont entraînés sur des volumes massifs de code et de bonnes pratiques.

Cela favorise souvent des solutions convergentes.

Cette homogénéisation possède des avantages :

- cohérence ;
- lisibilité ;
- standardisation.

Mais elle peut également réduire :

- l'originalité ;
- la diversité des approches ;
- l'innovation architecturale.

---

## Manifestation observable

Des équipes différentes produisent progressivement des solutions très similaires.

---

## Question clé

À partir de quel point la standardisation devient-elle une limitation ?

---

# Réduction des débats techniques

## Description

Une partie des apprentissages d'une équipe provient de ses désaccords.

Les discussions techniques obligent à :

- argumenter ;
- justifier ;
- expliciter les hypothèses.

Lorsqu'une réponse semble immédiatement disponible, certains débats disparaissent.

---

## Manifestation observable

Le groupe adopte rapidement la première proposition du LLM.

---

## Question clé

Les discussions ont-elles disparu parce qu'elles étaient inutiles ?

Ou parce qu'elles sont devenues inconfortables ?

---

# Amplification des bonnes pratiques

## Description

L'IA permet de diffuser rapidement :

- des patterns ;
- des conventions ;
- des pratiques de qualité.

Une équipe junior peut bénéficier immédiatement d'un niveau de conseil autrefois réservé à des experts.

---

## Manifestation observable

Amélioration rapide :

- des tests ;
- de la documentation ;
- du refactoring ;
- des conventions.

---

## Question clé

L'IA agit-elle ici comme un multiplicateur d'expertise ?

---

# Amplification des mauvaises pratiques

## Description

Le même mécanisme fonctionne dans l'autre sens.

Une mauvaise hypothèse acceptée par le groupe peut être propagée extrêmement vite.

---

## Manifestation observable

Une erreur est reproduite à grande échelle.

---

## Question clé

L'équipe conserve-t-elle des mécanismes de validation indépendants ?

---

# Érosion du bus factor

## Description

Le bus factor mesure combien de personnes doivent disparaître pour mettre un projet en danger.

L'IA peut améliorer ce facteur :

- documentation plus abondante ;
- onboarding plus rapide ;
- transfert de connaissances facilité.

Mais elle peut aussi le dégrader.

Si personne ne comprend réellement les décisions prises, le système devient dépendant :

- des prompts ;
- des historiques ;
- des agents.

---

## Question clé

L'équipe devient-elle plus résiliente ?

Ou simplement dépendante d'un nouvel acteur ?

---

# Vitesse d'équipe vs compréhension d'équipe

## Description

Une équipe peut augmenter fortement sa capacité de livraison.

La question est :

la compréhension progresse-t-elle au même rythme ?

---

## Manifestation observable

Le nombre de fonctionnalités livrées augmente.

La maîtrise du système n'augmente pas nécessairement.

---

## Question clé

La vitesse collective cache-t-elle une stagnation cognitive ?

---

# Émergence d'une expertise augmentée

## Description

C'est probablement le scénario le plus prometteur.

L'IA ne remplace pas les experts.

Elle leur permet :

- d'aller plus vite ;
- d'explorer davantage ;
- de partager leur savoir plus efficacement.

Dans ce modèle, l'IA devient un multiplicateur.

---

## Manifestation observable

Les experts deviennent plus accessibles.

Le niveau général de l'équipe augmente.

---

## Question clé

L'IA réduit-elle l'expertise ?

Ou démocratise-t-elle l'accès à l'expertise ?

---

# Fragmentation des modèles mentaux

## Description

Avant l'IA, plusieurs développeurs construisaient progressivement une compréhension commune du système.

Avec les assistants personnels, chacun dialogue avec son propre conseiller.

Deux développeurs peuvent désormais :

- travailler sur la même fonctionnalité ;
- recevoir des explications différentes ;
- adopter des hypothèses différentes.

---

## Manifestation observable

Les représentations du système divergent.

---

## Question clé

L'équipe possède-t-elle encore un modèle mental partagé ?

---

# Le nouveau rôle des seniors

## Description

Historiquement, les seniors étaient souvent :

- des sources de connaissance ;
- des référents techniques ;
- des mentors.

Lorsque l'information devient abondante, leur valeur se déplace.

Ils deviennent davantage :

- arbitres ;
- facilitateurs ;
- validateurs ;
- concepteurs de systèmes ;
- gardiens des compromis.

---

## Question clé

La valeur du senior réside-t-elle encore dans ce qu'il sait ?

Ou dans sa capacité à juger ?

---

# Le paradoxe collectif

L'IA peut simultanément :

- renforcer la collaboration ;
- affaiblir la collaboration.

- diffuser l'expertise ;
- diluer l'expertise.

- accélérer les équipes ;
- fragiliser les équipes.

- réduire les silos ;
- créer de nouveaux silos.

Comme pour les phénomènes individuels, l'issue dépend moins du modèle que de la manière dont l'organisation choisit d'intégrer l'IA dans ses pratiques.

---

# Hypothèse finale

L'avantage compétitif durable ne viendra probablement pas des équipes qui utilisent le plus l'IA.

Il viendra davantage des équipes qui sauront :

- conserver leur intelligence collective ;
- préserver leur capacité de jugement ;
- utiliser l'IA comme multiplicateur de compréhension plutôt que comme substitut à la compréhension.

L'enjeu n'est donc pas uniquement de créer des ingénieurs augmentés.

L'enjeu est de créer des équipes augmentées.
