# 01 · Phase delivery

Horaires → [`planning.md`](planning.md).
**Posture réelle (tes notes, jamais dite) : délestage cognitif → pôle aliéné.**
Le même geste qu'en phase méta-cognition (lancer plusieurs agents en parallèle),
mais ici pour *ne pas penser*.

## 🎯 À faire passer (aux participants — la surface seulement)

- Vous êtes en équipe, sous **pression de delivery** : un backlog, des agents, le chrono.
- Seul critère : **ça marche et c'est démontrable**. Le **nombre d'items bouclés** compte.
- **Parallélisez sans retenue** : lancez plusieurs agents à la fois, ouvrez plusieurs tickets en même temps, abattez le maximum.
- Allez vite, faites-vous confiance, **enchaînez**.

> ⚠ Ne dis **rien** de plus. Le but caché (provoquer la capitulation) reste secret —
> le backlog fait le reste. Pour eux, c'est juste un sprint réaliste.

## Consigne (à l'oral, + remise du `backlog.md`)

> On vient de vous embaucher dans l'équipe qui maintient Conduit.
> Voici le backlog : descendez-en un maximum avec l'IA.
> **Lancez plusieurs agents en parallèle** si ça vous fait gagner — tout est permis.
> Seul critère : que **ça marche et que ce soit démontrable**.
> C'est le nombre d'items bouclés qui sera mesuré.

## Pendant que ça tourne, observe

- le débit, et **combien d'agents tournent en parallèle** ;
- **le moment où la revue décroche** ;
- les questions à l'agent qui disparaissent ;
- la sensation affichée : « ça avance ».

## Ce que tu vas voir (et qui est voulu)

Presque tout le monde prend le **chemin naïf** : un booléen `draft/published`,
« ça marche » — sans penser la visibilité. Le brouillon fuite dans le feed, par
tag, dans le profil, via l'URL directe.

Le naïf n'est **pas faux** : il satisfait le brief. C'est exactement le piège.
**Ne moralise pas** : sous pression de débit, c'est légitime.
